Observability & Monitoring Platform

Project Overview

Designed and implemented a centralized observability platform for multi-tier production systems at TCS. The platform consolidates metrics, logs, and alerts from across the infrastructure into a unified view, enabling faster incident detection and response. By integrating Azure Monitor, Grafana dashboards, and the ELK Stack (Elasticsearch, Logstash, Kibana), the system significantly improved the team's ability to detect anomalies before they impacted end users — reducing unplanned downtime by 20%.

Technologies Used

Azure Monitor Log Analytics Elasticsearch Logstash Kibana Grafana Prometheus

Key Features

  • Centralized dashboards in Grafana and Kibana for real-time system visibility across all production environments
  • Threshold-based alerting rules in Azure Monitor to trigger notifications before SLA breaches
  • Log aggregation pipeline using Logstash feeding into Azure Log Analytics for unified log querying
  • Proactive anomaly detection that reduced unplanned downtime by 20%
  • Issue detection speed improved by 40% compared to manual monitoring approaches
  • Custom alert suppression logic to reduce alert fatigue during planned maintenance windows

Linux-Based Load Balancer (NGINX)

Project Overview

Designed and configured an NGINX-based Layer 7 reverse proxy and load balancer for a microservices application running on Linux. The setup replaced a manual traffic management approach and enabled automated rolling deployments, active health checks, and weighted round-robin distribution across backend nodes. The result was a significant improvement in application availability, reliability, and fault tolerance — eliminating single points of failure across the service tier.

Technologies Used

NGINX Linux (RHEL/Ubuntu) Bash Scripting DNS Configuration Firewall Rules

Key Features

  • NGINX configured as a Layer 7 reverse proxy with SSL termination for secure inbound traffic
  • Weighted round-robin load balancing distributing traffic across multiple backend instances
  • Active health checks automatically removing unhealthy nodes from the rotation
  • Rolling deployment support — new instances added to the pool without downtime
  • Firewall rules and DNS resolution configured for controlled, secure communication
  • Eliminated single points of failure, improving overall application availability and reliability
